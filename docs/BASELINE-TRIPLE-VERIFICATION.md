# Baseline triple-verification report — 18 July 2026

This report records the independent three-method recheck of the 25 active baseline findings, `BUG-001`, `BUG-003`–`BUG-025` and `BUG-027`. It is part of the existing audit package, not a separate audit-of-audit. Every prior claim was treated as untrusted and checked only against the supplied source snapshots.

## Result

- Static runtime traces: **25/25 passed**. Every quoted source excerpt was compared against the supplied trees by [`EVIDENCE-EXCERPT-VALIDATION.py`](EVIDENCE-EXCERPT-VALIDATION.py).
- Executable reproductions: **25/25 findings reproduced** by 26 focused tests in [`REVERIFICATION-CHECKS.py`](REVERIFICATION-CHECKS.py). `BUG-013` has two independent executable variants. The same suite also contains negative controls for archived `BUG-002` and reclassified `BUG-026`, for **28 passed** total.
- Active falsification: **25/25 passed** by [`BASELINE-FALSIFICATION-CHECKS.py`](BASELINE-FALSIFICATION-CHECKS.py), which searches for wrappers, aliases, rollback paths, normalization, locks, ownership tokens, fallbacks, unreachable states and documented contracts that could invalidate or narrow the reproductions.

Captured outputs: [`REVERIFICATION-CHECK-OUTPUT.txt`](REVERIFICATION-CHECK-OUTPUT.txt) and [`BASELINE-FALSIFICATION-CHECK-OUTPUT.txt`](BASELINE-FALSIFICATION-CHECK-OUTPUT.txt).
